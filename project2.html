Image Processing for Tomato Quality Analysis

Year 2025
Platform Python & OpenCV

How it Made?

As the final project for Robotic Tech course, I developed an image processing system to analyze the physical quality parameters of tomatoes. Building a comprehensive GUI application using PyQt5 and Qt Designer, which was integrated with OpenCV for real-time image processing. The system was designed to evaluate tomato quality based on color, size, and sweetness level (Brix) without using destructive testing methods.

Captured digital images of tomatoes with different ripeness levels and processed them using HSV color space segmentation to isolate the fruit from the background. I implemented adaptive thresholding and morphological operations to improve detection accuracy. The system automatically estimates physical parameters, including area, weight, dimensions, and Brix content, based on pixel analysis and RGB color values.

Technologies Used

Python OpenCV PyQt5 Qt Designer NumPy CSV Data Processing

The system demonstrated strong correlations between pixel-based measurements and actual physical parameters. Achieved an R² value of 0.936 for area prediction and 0.961 for weight estimation, indicating that pixel count is a reliable indicator for these parameters. The most significant finding was the relationship between RGB color values and Brix content, where the green component showed the strongest correlation (R² = 0.919), making it the most reliable predictor for tomato sweetness levels.

Validation tests comparing predicted values with actual measurements showed that the model performed best for Brix and dimensional parameters. While some deviations occurred in weight and area predictions for certain samples, the overall trend accuracy was maintained. This non-destructive testing method proved to be significantly faster and more consistent than manual evaluation, reducing analysis time while improving objectivity in quality assessment.
